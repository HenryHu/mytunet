运行 MyTunet for Windows 需要什么环境？
您的电脑需要运行 Windows 2000/XP/2003 等操作系统。Windows 下 MyTunet 会自动检测是否安装了 WinPCap ，如果没有安装， MyTunet 会自动安装必需的组件。

使用时请注意：官方 TUNet 或 MyTunet 只能选用一个，不要同时运行，否则会引起冲突。

如何安装 MyTunet for Windows？
与官方 Tunet 不同，MyTunet 无需安装。只需到 MyTunet 维护网站下载压缩包文件，解压缩到任何目录后即可运行。当然，我们建议您将它放在 C:\Program Files\MyTunet 中。

今后 MyTunet 如需升级，请将旧版本文件删除，然后将新版本文件解压至原处即可。

MyTunet 需要向注册表中记录少量信息（仅限于您的帐号、设置等），因此并不是严格意义上的绿色软件。

MyTunet for Windows 安全吗？

所有网络通信，MyTunet 与官方 Tunet 完全一致。在网络通信的数据流上， MyTunet 和官方 Tunet 的安全性完全等价。

保存密码登录时，MyTunet 并不保存用户的原始密码。原始密码在第一次被输入后就被立刻从内存中销毁，整个登录过程不再需要原始密码（官方 Tunet 也是这么做的）。当然， MyTunet 必须保留通过原始密码生成的一些信息（官方 Tunet也是这么做的）。 MyTunet 把这些信息通过 Windows Crypt 加密系统进行加密，加密方式是与当前 Windows 用户相关联的。也就是说，只有当前的 Windows 用户才可 以获得这些敏感信息，并可保证这些信息不被恶意用户窃取。

疑难解答： MyTunet for Windows 的状态图标具体是什么含义？

灰色的花朵 : 无可用状态信息。这时候 MyTunet 处于空闲状态。

红色的“8”: 当前正在进行与 802.1x 端口认证有关的操作。

紫色的花朵 : 当前正在进行与 打开校园连网权限 有关的操作。

绿色的花朵 : 当前正在进行与 打开国内（受限）连网权限 有关的操作。

黄色的花朵 : 当前正在进行与 打开国际（无限制）连网权限 有关的操作。

　

如果图标稳定显示（无闪烁），表示已经完成了这个操作。

如果图标在闪烁，则表示这个操作正在进行中。

操作指南：如何使用多帐户快速切换功能？

首先明确几个 MyTunet 里的概念：

登录信息：用户登录所需要的用户名/密码等各种资料。

用户帐户：一个帐户保存着一个套登录信息。包括用户名、密码、登录方式等等。

当前帐户：就是在 MyTunet 主界面里显示的一系列登录信息。注意！当前帐户并不是一个永久的帐户。每次选不同的用户帐户 登录后，当前帐户的信息都是这个用户帐户的信息。而且，修改当前帐户的信息并不改变任何用户帐户信息。简单的说，当前帐户只是进行最后一次 登录的用户帐户的一个拷贝。这个问题想说明白可能不容易，但只要你试一下就明白了，呵呵。

　

好了，切入正题：

1、添加/删除/修改 用户帐户。

   点“配置”按钮，进入“配置”对话窗。其中“帐户列表”就是用于管理用户帐户的。

   建立新帐户：在“帐户”输入框里输入帐户名（这个随便了，可以自己起名字）。然后输入“用户名”“密码”等信息，并点“保存”，这个用户帐户就建立了。

   删除帐户：在“帐户”下拉列表里选一个想删除的用户帐户，然后点“删除”，就这么简单。

   修改帐户：在“帐户”下拉列表里选一个想修改的用户帐户，然后修改所需要修改的信息，修改完后千万别忘了点“保存”哦！

2、快速切换帐户登录。

   方法一：在Windows任务栏托盘区（就是右边的那些图标区，用鼠标右键点  MyTunet  的图标，在“帐户列表”里选一个就可以了。

   方法二：在 MyTunet 主界面里也可以切换登录帐户。点“用户名”下拉框，然后可以看到一些帐户列表，选一个，则  MyTunet  会把选择的这个用户帐户的登录信息拷贝成当前帐户。然后再点登录就可以了。

    两种方法有什么不同？如果你已经登录了，方法一不需要注销就可以切换。方法二你必须先点“注销”，注销后才能再“登录”。

如果你还有不明白的地方可以联系我们的技术支持人员。我们将尽最大努力帮你解释清楚。

操作指南：如何把 MyTunet 安装为服务？

首先解释一下什么是服务（如果您很熟悉，请跳过这一段）：服务程序是一种特殊的 Windows 程序，它可以在系统刚启动的时候就被运行。也就是说，哪怕用户还没 登录Windows，只要开机，服务程序就会自动运行。这种程序一般用于提供网络服务等，所以就叫“服务（Service）”了。

 MyTunet 目前支持安装为一个服务，其服务程序是  MyTunetSvc.exe，安装成功后服务名是“ MyTunetSvc”。
当然，您并不需要知道它到底是如何工作的，在 MyTunet 的主界面中，您可以轻松完成对 MyTunet 服务的各种操作。

先说明一个基本问题： MyTunet 的普通应用模式和服务模式是不能同时启用的。也就是说，如果你安装了 MyTunetSvc 服务，那么 MyTunet 的所有操作都会依靠服务来进行，而不再是在主程序里进行操作了。

并且，安装  MyTunetSvc 服务后，与普通模式会有以下不同：

・开机后就自动连网，而不是登录 Windows 运行 MyTunet 程序后才连网。
・不再支持“最低余额警告保护”功能。
・对登录认证的各种重试操作都不再依靠用户设置，而由服务自己决定。
・MyTunet 主程序只起到“控制器”的作用，即只负责控制服务的运行，而不再亲自过问连网事宜。
・仅退出 MyTunet 主程序并不会中断当前的连网，想中断当前连网，必须点击主程序上的“注销”按钮，或停止 MyTunetSvc 服务。

安装 MyTunetSvc服务的操作很简单，点“配置”，再点“安装 MyTunet 服务”即可。服务安装后，即可在 MyTunet 主程序中对服务进行控制。

并且，只要运行了  MyTunet  主程序，服务会把信息反馈给主程序。如果 MyTunetSvc 服务没找到正在运行的 MyTunet 主程序，它将忽略反馈信息。

说明！除非您确实需要服务所提供的这些功能，否则强烈建议您不要采取服务模式来进行连网！

校园网使用常见问题与解决方案

这些问题是我根据以往很多朋友们问过的问题总结的。希望对大家有帮助。

0. 出现问题，请以官方的公告和说明为准。“清华大学BBS” 的 THUNet 版有关于网络问题的详细解答。

我这里只是列了一些那里没有的内容。

1. 如何关闭Windows XP SP2 默认的802.1x端口验证和防火墙（如果不关闭，有可能影响上网）？

打开网络和拨号连接：

有三条路可以走，任选其一：
一：打开“控制面版”->“网络和拨号连接”
二：在桌面上或开始菜单里，右键点“网络邻居”->“属性”
三：任务栏又下角又键点那个网络图标，选“状态”->“属性”

现在在网卡上点右键，选“属性”，
在第二个标签里，取消那个802.1x的身份验证选项（这个一定要关，否则与清华的登录冲突）。
在第三个标签里，选“禁用防火墙”（一般来说这个不时必须的，但如果登录有问题，可以关了试一下）。

2. 有的时候网络忽然断掉就再也连接不上了？

答：可以重新启动。或者，把网线拔掉重新插一下。

3. 什么是 802.1x 端口认证？清华校园网登录需要哪些验证（紫荆公寓区）？

答：802.1x 端口认证是一种网络协议。用于确认用户身份后再给用户打开网络连接。在紫荆公寓区，经过802.1x 端口认证，打开校园内的连网权限。然后再通过网络中心服务器，打开国内/出国 连网权限。

 MyTunet 的登录记录里把 802.1x 端口认证的记录标记为“[802.1x]”，把打开联网权限的记录标记为“[tunet]”

4. 为什么我过了 802.1x 端口认证，但打开连网权限时提示网络出错？
　　或者无法连接一些原来可以连接的 FTP ？

答：请确保：
(1)没有防火墙干扰（可以先把防火墙都关了再试一下，特别是Windows XP SP2自己带了一个防火墙。
(2)你的网卡配置信息是正确的。标准的网卡配置信息应该如下（紫荆公寓区）：

IP地址 59.66.254.43 我这是随便写了个例子，为了说明问题，不要深究，呵呵
子网掩码 255.255.255.0 注意！由于59.*.*.* 是 A 段 IP，因此 Windows 默认的子网掩码是255.0.0.0，一定要改过来！
默认网关 59.66.254.1 注意！网关的前三段与你的IP是一样的！
DNS服务器 166.111.8.28 , 166.111.8.29 这个没什么好说的，标配

说明：我遇到 n 多人把 网关 或 IP地址 之类写错了，就是上不去网。其实本来很简单的一件事，输入的时候小心点就 ok 了。

5. 我连网后几分钟就断， 然后就连不上了，但重新启动就好了。

答：很可能是网卡有问题。特别是一些集成的网卡， 总是工作不正常。如果你用的恰好是集成网卡，建议你借一块普通网卡（当然是最好能借到 Realtek 8139 了，呵呵） 来试试。如果问题解决，恭喜，买块网卡去吧。如果没解决…… 或许就是网络问题了。

说明：我已经帮两个同学换网卡了，都是换了就好，郁闷吧？

6. 无论是用 Tunet 还是 MyTunet ，均显示登录成功，但仍然无法访问校外网站，怎么办？

答：这是由于学校中心服务器没及时处理你的连网请求造成的。可以用以下方法解决：去 https://usereg.tsinghua.edu.cn/ 打开连网权限即可。
